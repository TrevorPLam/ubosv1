> **Project:** AI Command Center вҖ” Frontend UI Only
>
> Build the complete React/TypeScript frontend for a local-first multi-agent AI orchestration platform. This is a **UI shell only** вҖ” do not create any backend, server, database, authentication system, or deployment configuration. All data will come from a pre-existing FastAPI backend via HTTP and Server-Sent Events (SSE). Your job is to build the visual interface and wire it to that API contract.
>
> ---
>
> **Tech Stack вҖ” use exactly these, nothing else:**
> - React 18 + TypeScript
> - Vite (not Next.js, not CRA)
> - Tailwind CSS
> - shadcn/ui component library
> - TanStack Query (React Query v5) for all server state
> - Zustand for UI-only state (command palette open/closed, sidebar state)
> - React Router v6 for page routing
> - No Replit-specific packages, no Replit Auth, no Replit DB
>
> ---

> - `/chat` вҶ’ ChatInterface

> - `/analytics/audit` вҶ’ AuditLogPage
> - `/memory` вҶ’ KnowledgeBasePage
> - `/integrations` вҶ’ IntegrationsPage
> - `/settings` вҶ’ SettingsPage
> - `/settings/export` вҶ’ ExportImportPage
>
> ---
>
> **API Contract вҖ” use these mock endpoints (hardcode base URL as `http://localhost:8000`):**
> - `GET /api/v1/health` вҶ’ system health status
> - `GET /api/v1/agents` вҶ’ list of agents with status
> - `GET /api/v1/attention-queue` вҶ’ list of pending approval items
> - `POST /api/v1/attention-queue/{id}/approve` вҶ’ approve a decision
> - `POST /api/v1/attention-queue/{id}/reject` вҶ’ reject a decision
> - `GET /api/v1/chat/{thread_id}` вҶ’ chat message history
> - `POST /api/v1/chat` вҶ’ send a message, returns `thread_id`
> - `GET /api/v1/chat/{thread_id}/stream` вҶ’ SSE stream of agent tokens and events
> - `GET /api/v1/cost-analytics` вҶ’ 5-layer cost breakdown
> - `GET /api/v1/audit-log` вҶ’ paginated audit entries
> - `GET /api/v1/memory` вҶ’ project anchor + memory items
> - `GET /api/v1/mcp-servers` вҶ’ list of MCP integrations with trust tier
>
> Use realistic mock/placeholder data everywhere so the UI is fully navigable without a running backend.
>

> **Visual Design:**
> - Dark theme by default, with a subtle light mode toggle in settings
> - Left sidebar navigation, main content area, right panel for Attention Queue
> - Agent status uses five states: `idle` (grey), `thinking` (blue, animated pulse), `running-tool` (teal), `awaiting-approval` (amber), `error` (red)
> - Every status indicator must use **both color and an icon shape** вҖ” never color alone
> - The ActivityFeed auto-scrolls but has a visible "Pause Updates" button
> - The CommandPalette opens with `вҢҳK` / `Ctrl+K` and searches across agents, pages, and recent actions
> - The AttentionQueue sits in a persistent right panel and shows a badge count when items are waiting
>
> ---
>
> **Accessibility Requirements:**
> - `ActivityFeed` container must have `role="log"` and `aria-live="polite"`
> - `CommandPalette` input must use `role="combobox"` with `aria-expanded` and `aria-activedescendant`
> - All modals must trap focus and use `role="dialog"` with `aria-modal="true"`
> - All buttons must have descriptive `aria-label` attributes
> - Tab order must follow visual reading order: sidebar вҶ’ main content вҶ’ right panel
>
> ---
>
> **Constraints вҖ” strictly do not:**
> - Create any Python files, FastAPI routes, or backend logic
> - Create any database schema or ORM models
> - Add any authentication or session management
> - Use Next.js, Remix, or any SSR framework
> - Add any Replit-specific packages or configuration
> - Create a `server.js` or any Node.js server
> - Generate Docker files, CI/CD configs, or deployment manifests
>
> The output must be a clean, exportable Vite + React project that can be dropped into a larger monorepo's `frontend/` directory with zero modifications.
